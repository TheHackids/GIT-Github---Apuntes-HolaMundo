Core.autocrlf

En windows cada vez que agregamos un salto de linea, agrega dos caracteres:

  • Carriage Return (CR)
  • Line Feed (LF)

Si trabajas en LINUX/MAC, solo se agrega line feed (LF)

Por lo tango, el usuario de WINDOWS va a tener que eliminar el CR de su código si quiere subir su código y si quiere descargar tendrá que agregar el caracter CR, para ello, Core.autocrlf tiene que tener un valor de true

Puede pasar que un usuario de LINUX/MAC por el editor de código que utilice o manualmente, agregue el caracter CR, en ese caso, la propiedad autocrlf debe contener input

Comandos

  • ls nos permite listar todos los archivos y carpetas de un directorio
  • pwd me indica en que directorio estoy
  • cd sirve para ir a cualquier carpeta
  • cd .. es para salir del directorio y nos manda a uno más arriba en la jerarquía
  • mkdir se utiliza para crear un directorio
  • git init crea un repositorio en GIT, para poder gestionar el proyecto
  • ls -a nos permite visulizar el respositorio que se ha creado con formato .git
  • code . nos permite abrir el repositorio en nuestro editor de código favorito (VISUAL STUDIO CODE para mí)
  • git status es para ver el estado de nuestro repositorio
  • git add sirve para añadir archivos al repositorio, NO USES EL PUNTO
  • git commit -m "" me permite comprometer los nuevos archivos, es decir, darle luz verde a la actualización
  • rm se utiliza para eliminar un archivo en la etapa computador, para luego cargarlo a la etapa de stage
  • git rm sirve para eliminar un archivo y colocarlo directamente en la etapa de commit
  • git restore --staged regresa un archivo de commit a la fase de stage
  • git restore es para recuperar un archivo de la fase stage al computador
  • mv nombreOriginal nombreFinal sirve para cambiar el nombre de un archivo y llevarlo a la fase stage
  • git mv nombreOriginal nombreFinal se utiliza para cambiar el nombre y enviarlo directamente a la fase de commit
  • git diff Nos muestra los cambios del archivo a detalle
  • git diff --staged Muestra todos los cambios que se encuentren en la etapa de stage
  • git log Muestra a detalle los commits realizados
  • git log --oneline Muestra un resumen de los commits realizados
  • cat nombreDelArchivo Nos muestra el contenido del documento

Ignorar documentos

Se debe crear el documento .gitignore y colocar los documentos que desees ignorar, dentro de este y luego hacer commit a .gitignore

Ramas o branches

Antes que nada, debes saber en que rama nos encontramos

Usamos el comando git branch para saber nuestra rama

Para crear una rama usamos git checkout -b nombreDeLaRama

Para traer las modificaciones de una rama a otra usamos git merge nombreDeLaRama
